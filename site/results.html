Failure notes

The hard cases were fake-lib and hidden-shape traps.

flake

Metadata shape

Both runs missed required app metadata on the flake task, showing how easy it is to satisfy visible structure while losing hidden attributes.

lib

Unavailable helpers

GPT used escapeShellArgs and getExe; Claude used concatStringsSep. The evaluator fake lib intentionally rejected those assumptions.

license

Wrong-but-plausible values

Both models chose mit for the Python packaging task, while the hidden evaluator expected asl20.

How to read this run

Useful evidence, with narrow claims.

Objective score

Hidden shell evaluators score the final worktree. No LLM judge decides whether a solution is good.

Single samples

Each model ran once, so the table reports observed outcomes—not confidence intervals or expected pass rates.

Matched task contract

Both runs used the same 10 tasks, prompt contract, and 240-second per-task timeout.
