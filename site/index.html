The agent receives a copied starter worktree, edits local files, exits, and then NixBench runs a hidden shell evaluator. Passing means the Nix expression survived evaluation, not just review.

What it measures

Nix mistakes that hide in plain sight.

eval

Semantic correctness

Tasks inspect flakes, modules, overlays, derivations, shells, fetchers, and attrsets with objective checks.

fake lib

Restricted environments

Evaluators use small fake builders and package sets so unavailable helpers, host paths, and overfit assumptions fail quickly.

diff

Agent behavior

Every run records logs, timing, diffs, task scores, and timeout state for reproducible comparisons.

Run model

A benchmark task is a small repository repair.

NixBench avoids isolated snippet prompts. The harness creates a temporary workdir, copies the starter files, runs the agent command there, and evaluates the resulting worktree after the agent exits.

  1. copy Starter files and prompt enter a clean temporary workdir.
  2. edit The agent reads NIXBENCH_PROMPT.md and changes local files.
  3. check Hidden evaluators score the final worktree.
  4. record Logs, diffs, JSON summaries, and timings are written under results/.
