The question

Does the final repository work?

The score comes from the tree an agent leaves behind, not from an explanation, a preferred coding style, or an LLM’s opinion.

eval

It must evaluate

Hidden checks inspect flakes, modules, overlays, derivations, shells, fetchers, attrsets, and issue reports.

fake lib

Shortcuts must fail

Constrained builders and package sets catch unavailable helpers, host paths, and solutions overfit to the visible example.

diff

Evidence is preserved

Runs retain logs, timing, diffs, scores, evaluator output, and timeout state so a result can be inspected.

Evaluation loop

One prompt in. One scored worktree out.

Each task is a small repository repair. The evaluator stays hidden until the agent has finished, keeping visible instructions separate from the scoring contract.

Read the benchmark design →
  1. copy Starter files and prompt enter a clean temporary workdir.
  2. edit The agent reads NIXBENCH_PROMPT.md and changes local files.
  3. check Hidden evaluators score the final worktree.
  4. record Logs, diffs, JSON summaries, and timings are written under results/.
